\begin{abstract}
We present \texttt{flatmem}, a constant-RAM content-addressable memory
substrate for online-learning digital organisms.  The architecture integrates
five mathematical primitives---Sparse Distributed Memory (SDM)
\cite{kanerva1988sparse}, Vector Symbolic Architecture (VSA) role-binding
\cite{plate1995holographic,kanerva2009hyperdimensional}, computed-identity
keys, read-time mean-removal \cite{mu2017all}, and traffic-class bank
separation---into a single substrate whose total bytes are \emph{fixed}
regardless of how much information is written.  A trained instance occupies
192~MB of substrate carrying co-occurrence statistics, IS-A taxonomy,
sensory grounding, verb-rotor arithmetic, transition probabilities, and
classification labels for a natural-language plus vision organism;  full
inference RAM is 233~MB.  We empirically demonstrate constant-RAM behaviour
up to 100K text exposures (vocabulary of 19{,}411 words), substrate-flatness
through cross-modal training, modality-blind extension to a vision channel
(90\% test accuracy on 8$\times$8 digit classification), and---most
critically---catastrophic-forgetting-free continual learning: 5 original
facts retained with $100\%$ recall through 5{,}000+ distractor writes
spanning four different channels.  We argue \texttt{flatmem} is the
universal flat-memory primitive for Artificial Life simulations, embedded
agents, and any system that needs lifelong learning within a bounded RAM
budget.  Code is open-source under MIT license at
\url{https://github.com/HitoshiFTW/Flatmem-Mura-ALife-Labs} and installable
via \texttt{pip install flatmem}.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Modern machine learning systems scale memory with data.  Embedding tables
in language models grow with vocabulary
\cite{mikolov2013distributed,pennington2014glove};  replay buffers grow
with corpus size;  vector databases grow linearly with stored items.
Biological brains do neither.  An adult human cortex contains roughly
$86 \times 10^9$ neurons and approximately $10^{14}$ synapses fixed from
development \cite{bartol2015nanoconnectomic};  a lifetime of experience
is encoded in the \emph{strengths} of synaptic connections within that
fixed matrix, not in newly allocated storage.  This gap has practical
consequences for online-learning systems in robotics, multi-agent
simulations, and edge deployment, where unbounded growth is unacceptable.

Existing approaches sit at extremes.  Static word-embedding tables
(\textsc{word2vec} \cite{mikolov2013distributed}, \textsc{GloVe}
\cite{pennington2014glove}, contextual variants such as BERT
\cite{devlin2019bert}) require an entry per token or sub-word unit
\cite{sennrich2016neural}---memory $\Theta(|V|)$ in vocabulary $|V|$.
Vector databases inherit this growth.  Transformer language models
\cite{vaswani2017attention,brown2020gpt3} compress vocabulary into hidden
layers but suffer \emph{catastrophic forgetting} under sequential training
\cite{mccloskey1989catastrophic,french1999catastrophic} without
expensive remediation (Elastic Weight Consolidation
\cite{kirkpatrick2017overcoming}, Memory Aware Synapses
\cite{aljundi2018memory}, gradient episodic memory
\cite{lopez2017gradient}, full retraining, or parameter-efficient
fine-tuning with LoRA \cite{hu2022lora}).  Sparse Distributed Memory
(Kanerva 1988 \cite{kanerva1988sparse}) provides a biologically grounded
fixed-capacity substrate but, in its classical binary form, supports only
single-channel autoassociative recall.

This work proposes a different design point.  The substrate is a
fixed-size counter bank.  Knowledge is \emph{superposed} across many
cells;  each cell participates in many memories.  Identity is
\emph{computed} from input strings or vectors rather than looked up in a
table.  Recall is \emph{reconstructive}---lossy and gist-based, like
human memory.  The result is a substrate that:

\begin{itemize}
  \item Learns online (single-write per fact, no gradients).
  \item Never forgets catastrophically (additive superposition).
  \item Has bounded RAM at any data volume.
  \item Runs on a laptop CPU (no GPU required).
  \item Hosts multiple channels in one substrate (text, taxonomy, sensory,
        arithmetic, transitions, classification).
  \item Is modality-blind (text and pixels coexist).
\end{itemize}

\paragraph{Contributions.}  The contributions of this work are:
\textbf{(1)}~the empirical discovery and structural resolution of
\emph{traffic-class bank separation} as a precondition for multi-channel
VSA-SDM substrates (Section~\ref{sec:arch});
\textbf{(2)}~the dynamic adaptation of the static all-but-the-top
common-direction removal of Arora et al.\ \cite{mu2017all} to live recall
within a content-addressable memory, yielding a $14\times$ improvement in
related/unrelated pair separation (Section~\ref{sec:results});
\textbf{(3)}~zero-storage computed-identity keys with a whole-word phasor
weight parameter that eliminates lexical-overlap collisions
(Section~\ref{sec:arch});
\textbf{(4)}~the systemic integration of these into a complete
online-learning, multi-channel, modality-blind organism within a fixed
192~MB budget;  and
\textbf{(5)}~empirical demonstration of catastrophic-forgetting-free
continual learning across four distinct channels with 5,000+ distractor
writes (Section~\ref{sec:no-forgetting}, Figure~\ref{fig:noforget}).

Source code, the four ALife integration example scripts, and the figure
generation scripts used in this paper are available at the project
repository under MIT license.

\section{Background}
\label{sec:background}

\subsection{Sparse Distributed Memory and the cerebellar model}
Kanerva (1988) \cite{kanerva1988sparse} introduced SDM as a mathematical
model of cerebellar function, building on Marr \cite{marr1969cerebellum}
and Albus \cite{albus1971theory}, with biological grounding from
Eccles, Ito, and Szent\'agothai \cite{eccles1967cerebellum,ito2008control}.
The substrate is a bank of $M$ hard locations with random binary addresses;
writing activates the subset of locations within Hamming radius $r$ of the
query;  reading averages the contents of activated locations.  Capacity is
bounded but graceful: information degrades smoothly as the substrate fills
\cite{joglekar2015capacity}.  Frady and Sommer
\cite{frady2019robust} extended SDM to complex phasor space (Threshold
Phasor Associative Memory, TPAM), gaining native support for continuous
quantities and significantly higher capacity than binary SDM.  Hopfield
networks \cite{hopfield1982neural,krotov2016dense,ramsauer2020hopfield}
form a parallel lineage of associative memory at higher capacity but
different operational semantics (energy descent rather than location-set
averaging).

\subsection{Vector Symbolic Architectures}
Plate's Holographic Reduced Representations (HRR)
\cite{plate1995holographic}, Kanerva's Spatter Codes
\cite{kanerva2009hyperdimensional}, Gayler's framing
\cite{gayler2003vector}, and Rachkovskij and Kussul's binary thinning
\cite{rachkovskij2001binding} established VSA as a family of distributed
representations where structured data (trees, graphs, key-value bundles)
can be encoded as flat vectors via \emph{binding} (typically circular
convolution or component-wise multiplication) and \emph{bundling}
(superposition).  Recent surveys by Schlegel et al.\
\cite{schlegel2022comparison}, Kleyko et al.\ \cite{kleyko2022survey},
and Ge \& Parhi \cite{ge2020classification} compare VSA variants and
applications.  Frady, Kent, Olshausen, and Sommer
\cite{frady2020resonator,kent2020resonator2} introduced Resonator
Networks for efficient factorization of bound structures.

\subsection{Common-direction removal in static embeddings}
Mu, Bhat, and Viswanath \cite{mu2017all} observed that pre-trained word
embeddings share a dominant ``common discourse'' direction that
suppresses semantic discrimination.  Removing the top principal components
(``all-but-the-top'') dramatically improves retrieval.  The technique is
presented as a static post-processing step for fixed embedding tables;
this work adapts it to live recall in a content-addressable memory
(Section~\ref{sec:results}).

\subsection{Continual learning and catastrophic forgetting}
McCloskey \& Cohen \cite{mccloskey1989catastrophic} and French
\cite{french1999catastrophic} characterised catastrophic forgetting in
connectionist networks.  Parisi et al.\ \cite{parisi2019continual} survey
contemporary continual-learning approaches.  Common mitigations include
weight-importance regularization
\cite{kirkpatrick2017overcoming,aljundi2018memory}, gradient episodic
replay \cite{lopez2017gradient}, and parameter-efficient adaptation
\cite{hu2022lora}---each adds machinery to the trained model.  This work
argues that the catastrophic-forgetting problem is an artefact of how
neural networks encode memory (in shared weights modified by gradient
descent);  a substrate built on \emph{additive} superposition over
\emph{sparse} hard locations bypasses the problem structurally.

\subsection{Hyperdimensional Computing in Artificial Life and neuromorphic hardware}
Yilmaz \cite{yilmaz2015symbolic} applied VSA-based reservoir computing
to cellular automata.  Ultra-lightweight edge robotics increasingly
leverages HDC for resource-constrained sensorimotor integration
\cite{rahimi2017hyperdimensional}.  Specialised neuromorphic hardware
including IBM TrueNorth \cite{merolla2014million} and Intel Loihi
\cite{davies2018loihi} provide platforms where flat memory primitives
can run with substantial energy advantages over GPUs;  Indiveri et al.\
\cite{indiveri2011neuromorphic} review the broader silicon-neuron
landscape.  The Hierarchical Temporal Memory tradition initiated by
Hawkins and Ahmad \cite{hawkins2016neurons} shares the high-sparsity
distributed-representation philosophy.

\subsection{Memory in deep learning systems}
End-to-end memory networks \cite{sukhbaatar2015end}, differentiable
neural computers \cite{graves2016hybrid}, and compressive transformers
\cite{rae2020compressive} address memory in deep architectures.  All
require gradient training and substantial parameter overhead, in
contrast to the gradient-free, content-addressable design pursued here.

\section{Architecture}
\label{sec:arch}

\subsection{Notation}
We work in complex-valued phasor space.  A hypervector (HV) is a vector
in $\mathbb{C}^d$ where each component has unit magnitude ($|h_j| = 1$).
The default dimension is $d = 512$.  Similarity is the normalized
real-part inner product:
$\cos(a, b) = \operatorname{Re}\langle a, b \rangle / d$.  The
element-wise complex multiplication $a \odot b$ is the VSA \emph{bind}
operation;  componentwise unit-phasor renormalization is denoted
$\operatorname{renorm}(\cdot)$.

\subsection{Computed identity}
Item identity is not stored.  For a string $w$, the HV is computed as
\begin{equation}
  \mathrm{key}(w) = \operatorname{renorm}\!\Big(
    \!\!\sum_{tg \in \mathrm{trigrams}(\#w\#)} P(tg)
    \;+\; \alpha \cdot P(\mathrm{WORD}{:}w)
  \Big),
  \label{eq:key}
\end{equation}
where $P(\cdot)$ is a seeded random phasor projection
$P(\mathrm{tag}) = \exp(i \cdot \mathrm{uniform}(-\pi, \pi)_d)$ with
$\mathrm{seed} = \mathrm{SHA{-}256}(\mathrm{tag})$.  Character trigrams
give morphological similarity for out-of-vocabulary (OOV) inputs;  the
whole-word phasor weight $\alpha = 4$ ensures distinct strings have
near-orthogonal keys regardless of character overlap.  Empirically,
$\cos(\mathrm{key}(\text{little}), \mathrm{key}(\text{litttle}))$ drops
from $0.85$ (trigrams alone, $\alpha = 0$) to $0.09$ with $\alpha = 4$
(see Section~\ref{sec:ablation}).  Per-item storage cost: zero bytes;
the regenerable cache is for performance only.

\subsection{Sparse Distributed Memory substrate}
The substrate consists of two arrays: hard-location addresses
$H \in \mathbb{C}^{M \times d}$ (fixed random unit phasors, regenerable
from seed) and learned counters $C \in \mathbb{C}^{M \times d}$
(initialized to zero, written via Hebbian accumulation).  We store the
conjugate $H^* = \overline{H}$ explicitly for fast similarity computation.

\begin{algorithm}[t]
\caption{SDM write and read}
\label{alg:sdm}
\begin{algorithmic}[1]
  \Function{Write}{$\mathrm{addr},\, \mathrm{data}$}
    \State $\mathrm{sims} \gets \operatorname{Re}(H^* \cdot \mathrm{addr})$
    \State $\mathrm{idx} \gets \mathrm{argpartition}_{\text{top-}k}(\mathrm{sims})$
    \State $C[\mathrm{idx}] \mathrel{+}= \mathrm{data}$
  \EndFunction
  \Function{Read}{$\mathrm{addr}$}
    \State $\mathrm{sims} \gets \operatorname{Re}(H^* \cdot \mathrm{addr})$
    \State $\mathrm{idx} \gets \mathrm{argpartition}_{\text{top-}k}(\mathrm{sims})$
    \State \Return $\operatorname{renorm}\big(\textstyle\sum_{m \in \mathrm{idx}} C[m]\big)$
  \EndFunction
\end{algorithmic}
\end{algorithm}

Total substrate bytes: $2 \cdot M \cdot d \cdot 8$ (complex64).  With
$M = 16{,}384$, $d = 512$, $k = 64$ this is 128~MB per bank.
\textbf{The substrate size is independent of how much data is written};
writes only accumulate into existing counters.

\subsection{VSA role-binding for multi-channel storage}
To store multiple relation types in one substrate, item keys are bound
with role phasors:
\begin{equation}
  \mathrm{addr}(\mathrm{item}, \mathrm{role})
    = \mathrm{key}(\mathrm{item}) \odot R[\mathrm{role}].
  \label{eq:bind}
\end{equation}
For unit phasors,
$\langle x \odot a, x \odot b \rangle = \langle a, b \rangle$, so two roles
with random $R$ vectors yield address vectors with the same cosine
similarity as the underlying role vectors themselves.  Random role
phasors are near-orthogonal in $\mathbb{C}^d$, producing essentially
disjoint top-$k$ activation sets at the substrate level---zero
address-level interference.  We use six base roles:
\texttt{cooccur} (co-occurrence context), \texttt{next} (transition
successor), \texttt{isa} (taxonomy), \texttt{sensory} (perceptual
category), \texttt{property} (attribute), \texttt{verb} (scalar
coefficient).  A seventh role \texttt{class} is added for vision
experiments.  Additional roles can be registered at runtime by
generating a fresh random phasor.

\subsection{Traffic-class bank separation}
A naive single-substrate implementation fails empirically.  High-traffic
channels (co-occurrence: millions of writes) and low-traffic channels
(IS-A: tens of writes) \emph{share hard locations even with orthogonal
addresses}, because top-$k$ activation draws from the same pool.  The
accumulated magnitude at a high-traffic location dominates by orders of
magnitude (in our 5K-story experiment: $\|C[m]\| \approx 2{,}000$ for
hot co-occurrence cells vs.\ $\approx 22$ for a single IS-A write).
Sparse signals drown.

The fix is two separate fixed banks partitioned by expected traffic
class.  The \texttt{cooccur} and \texttt{next} roles write to a dense
bank;  \texttt{isa}, \texttt{sensory}, \texttt{property}, \texttt{verb},
\texttt{class} write to a sparse bank.  Each is independently
fixed-size.  Total RAM stays constant ($128 + 64 = 192$~MB at default
parameters).  This is, to our knowledge, a discovery not present in
current VSA literature, and it is a precondition for hosting both
high-write-rate continuous channels (co-occurrence statistics) and
low-write-rate symbolic channels (taxonomic facts) in a single
content-addressable memory.

\subsection{Read-time mean-removal}
Recalled vectors are dominated by a common direction---the ``general
discourse'' of the corpus, analogous to Mu, Bhat, and Viswanath's static
finding \cite{mu2017all}.  We adapt the technique to live SDM recall:
\begin{align}
  V &= \mathrm{SVD}_{\text{top-}r}\!\big([\,\mathrm{read}(w_i)\,]_{i=1}^{S}\big), \\
  \mathrm{clean}(w) &= \operatorname{renorm}\!\Big(
    \mathrm{read}(w) - \!\!\sum_{v \in V} \langle v, \mathrm{read}(w) \rangle \cdot v
  \Big),
  \label{eq:meanremove}
\end{align}
where $S$ is a sample of recently exposed items (default $S = 2{,}000$).
With $r = 1$, separation between related and unrelated word pairs jumps
from $0.026$ to $0.371$ (a $14\times$ improvement) on a 15K-story
TinyStories corpus.  The adaptation is dynamic---it tracks the common
direction as the substrate evolves.  Section~\ref{sec:results} reports
the full ablation.

\subsection{Reinforcement and Hebbian accumulation}
Multiple writes to the same address accumulate magnitude.  Cleanup
against a candidate vocabulary picks the highest-magnitude target.  No
separate write-count dictionary is required;  \textbf{magnitude is the
count}, biologically aligned with synaptic potentiation
\cite{bartol2015nanoconnectomic}.  Direct fact injection uses the API
\texttt{assert\_relation(item, role, target, n=N)} which writes $N$ times
without modulation.  Empirically $N \in [20, 50]$ suffices to outweigh
ambient noise from co-occurrence writes;  the optimal $N$ depends on the
relative write traffic between the asserted fact channel and competing
channels in the same bank.

\section{Implementation}
\label{sec:impl}

\subsection{Software stack}
The reference implementation is pure Python plus NumPy.  Total lines of
executable code:  under 500 in the core (\texttt{flatmem/core.py}) and
under 100 in the encoders module (\texttt{flatmem/encoders.py}).  No
backpropagation, no autograd, no GPU dependency, no external services.

\subsection{Tokenization}
Text is tokenized by lowercasing, replacing non-alphanumeric characters
(retaining apostrophes) with whitespace, and splitting on whitespace.
Out-of-vocabulary inputs are handled gracefully because identity keys are
computed from character trigrams (Section~\ref{sec:arch}); no fallback
UNK token is required.

\subsection{Helper encoders}
The encoders module provides:
\begin{itemize}
  \item \texttt{scalar\_phasor($v$)}: encodes a real scalar as a
    fractional-power phasor along a fixed bipolar axis.
    Useful for rewards, concentrations, joint angles, time.
  \item \texttt{position\_phasor($x, y$)}: encodes a 2D coordinate by
    binding two independent fractional-power axes.  Spatial proximity
    yields high cosine similarity, supporting topological generalization.
  \item \texttt{random\_projection\_encoder(vector)}: encodes an
    arbitrary dense numeric array via a seeded random projection.
    Used for image patches, sensor frames, and learned embeddings.
  \item \texttt{bind}, \texttt{unbind}, \texttt{bundle}, \texttt{permute}:
    primitive VSA operations.  \texttt{permute} (cyclic shift) is
    used for sequence-position encoding (e.g., genome ordering).
\end{itemize}

\subsection{Persistence and serialization}
Save and load use the standard Python \texttt{pickle} protocol with a
custom \texttt{\_\_getstate\_\_} that drops regenerable structures (the
hard-location address bank $H$, the per-word activation cache).  These
are recomputed deterministically from the substrate's seed on load.
A trained organism's checkpoint is dominated by the counter bank $C$
plus the small relational target sets, typically 160--200~MB regardless
of training duration.

\subsection{Caching and performance}
Two optional caches improve runtime: a per-word identity-key cache
inside \texttt{ComputedKey} (avoids recomputing SHA-256 hashes for
frequently seen items) and a per-(item, role) activation cache inside
\texttt{VSASDM} (avoids recomputing the top-$k$ matmul for repeated
writes to the same address).  Both caches are dictionaries of
\texttt{numpy} arrays;  both are explicitly excluded from
\texttt{\_\_getstate\_\_} since they are regenerable from the
deterministic key function and the fixed address bank.  Removing
either cache reduces RAM by a small constant amount at proportional
runtime cost;  neither affects the substrate's flatness claim because
both are O(vocabulary) in scale, dominated by the substrate's fixed
O($M \cdot d$) when vocabulary is large.

\section{Empirical Results}
\label{sec:results}

\subsection{Experimental setup}
Unless otherwise noted, experiments use $d = 512$, $M = 16{,}384$,
$M_{\mathrm{rel}} = 8{,}192$, $k = 64$, and 3K-100K story text exposures
drawn from the TinyStories dataset.  Vocabulary sizes after each
training scale are: $9{,}841$ at 10K stories, $12{,}658$ at 25K, $15{,}493$
at 50K, $17{,}648$ at 75K, $19{,}411$ at 100K (Heap's law with exponent
$\approx 0.55$).

\subsection{Substrate flatness at scale}

Figure~\ref{fig:flat} shows the substrate footprint (192~MB) compared
to a hypothetical embedding-table dictionary of the same dimension and
\texttt{complex64} dtype.  At 100K stories (17.3M tokens) the substrate
has absorbed a vocabulary of 19{,}411 distinct words with \emph{zero
growth in bytes}, while the equivalent $d{=}2048$ \texttt{complex64}
dictionary would require 303~MB.  This is the central engineering claim
of the substrate, validated across five training scales.

\subsection{Discrimination via mean-removal}

Figure~\ref{fig:separation}~(left) shows the impact of common-direction
removal rank $r$ on related-vs-unrelated pair separation at 15K stories.
With $r=0$ (raw recall) separation is 0.026---essentially noise because
all words share a dominant general-discourse direction.  At $r=1$
separation jumps to 0.371, a $14\times$ improvement.  Higher $r$ values
modestly reduce separation, suggesting the second-and-higher principal
components carry semantic signal that should not be removed.
Figure~\ref{fig:separation}~(right) shows separation holding at
$\sim 0.15$ across 25K to 100K stories, demonstrating the technique is
scale-stable.

\subsection{Catastrophic-forgetting-free continual learning}
\label{sec:no-forgetting}

We injected 5 original IS-A facts (\textit{aardvark}~$\to$~mammal,
\textit{lyrebird}~$\to$~bird, \textit{manatee}~$\to$~mammal,
\textit{penguin}~$\to$~bird, \textit{quokka}~$\to$~marsupial) with
$n{=}50$ reinforcement.  We then ``flooded'' the substrate with
distractors across four channels in sequence:
\begin{enumerate}
  \item +20 additional IS-A facts (sparse bank, same role)
  \item +5{,}000 co-occurrence text exposures (dense bank)
  \item +100 verb arithmetic observations (sparse bank, different role)
  \item +100 vision classifications (sparse bank, \texttt{class} role)
\end{enumerate}
After each distractor wave we re-queried the original 5 facts.  All 5
remained correctly recallable through every wave.  Additionally, all 20
distractor IS-A facts also remained recallable (no blocking;
accumulation is additive).  See Figure~\ref{fig:noforget}.

\begin{figure}[t]
  \centering
  \includegraphics[width=0.92\linewidth]{figures/fig2_no_forgetting.pdf}
  \caption{Original 5 IS-A facts retained across cross-modal distractor
  floods (5K+ writes across IS-A, text, verb, vision channels).  The
  ``naive sequential transformer'' bars are illustrative;  exact
  transformer behaviour depends on architecture, training regime, and
  whether replay or weight-protection methods are used.  \texttt{flatmem}'s
  no-forgetting is structural: superposition is additive, new writes
  activate different hard locations than old ones, nothing gets
  overwritten.}
  \label{fig:noforget}
\end{figure}

\subsection{Vision channel: modality-blind extension}

Adding pixel input required no architectural changes.  An 8$\times$8
greyscale image is flattened, projected via a fixed seeded random
matrix $P \in \mathbb{R}^{d \times 64}$ to a phase vector, and
exponentiated to yield a phasor HV.  Writing under the \texttt{class}
role with the label key as data trains a classifier;  cleanup against
$\{0, 1, \ldots, 9\}$ yields a prediction.  Figure~\ref{fig:vision}
shows results.  Critically, the same substrate that trained the
classifier still contains intact text co-occurrence, IS-A facts, and
verb-rotor arithmetic from a prior training phase, all queryable.

  \centering
  \includegraphics[width=0.65\linewidth]{figures/fig4_vision_results.pdf}

\subsection{Cross-channel composition}

Storing multiple relation types in one substrate enables cross-channel
chained recall.  Given $\mathrm{isa}(\textit{cat}) = \textit{mammal}$ and
$\mathrm{property}(\textit{mammal}) = \textit{warm}$, the chain
$\mathrm{property}(\mathrm{isa}(\textit{cat}))$ resolves to \textit{warm}
through two role queries against the same memory, with no explicit join.
Table~\ref{tab:chain} reports a small symbolic suite.

\subsection{Deployment footprint}

A trained flat-only organism (5K TinyStories, 5K math stories, clean
IS-A and sensory facts) checkpoints to 161~MB on disk.  Loaded into a
fresh process, total resident-set RAM is 233~MB:  192~MB substrate,
41~MB Python interpreter plus NumPy plus small caches.  This number is
stable across subsequent queries, indicating no leak.  Inference RAM
is the salient deployment number;  training-time RAM is higher due to
HuggingFace dataset library buffers, but that overhead is not present
at inference.

\section{Ablation Studies}
\label{sec:ablation}

\subsection{Whole-word phasor weight $\alpha$}
We ablate the whole-word phasor weight $\alpha$ in Equation~\ref{eq:key},
measuring lexical-collision rates on pairs of similar strings.
With $\alpha = 0$ (character trigrams alone),
$\cos(\mathrm{key}(\text{little}), \mathrm{key}(\text{litttle})) = 0.85$

phasor adds approximately $\alpha$ random unit components to the
trigram sum, so the contribution scales linearly;  $\alpha = 4$ was
chosen empirically and remains the default.  See Table~\ref{tab:wordweight}.

\subsection{Activation count $k$}
The number of hard locations activated per write ($k$) controls
substrate sparsity.  At fixed $M$, larger $k$ increases the per-write
``surface area'' and the crosstalk between unrelated writes; smaller
$k$ reduces crosstalk but also reduces the redundancy that protects
against location-level noise.  We found $k = 64$ (with $M = 16{,}384$,
a sparsity of $\approx 0.4\%$) optimal across the IS-A, vision, and
no-forgetting benchmarks;  $k \in [32, 128]$ produced qualitatively
similar results, while $k > 256$ degraded sparse-channel discrimination.

\subsection{Hard-location count $M$ and capacity}
The hard-location count $M$ sets the substrate's information capacity.
With $M = 16{,}384$ dense and $M_{\mathrm{rel}} = 8{,}192$ sparse banks,
we successfully held 19{,}411 word identities, $\sim 25$ explicit IS-A
facts, 8 verb-rotor coefficients, 9 sensory anchors, and 1{,}257
classified images simultaneously.  Doubling $M$ doubles the substrate
RAM linearly while approximately doubling effective capacity;  this is
the principal scaling lever for organisms that need to handle larger
vocabularies or more relations.

\subsection{Common-direction removal rank $r$}
Figure~\ref{fig:separation}~(left) reports the full sweep.  The
optimum is $r = 1$;  $r \geq 2$ removes useful signal in addition to
the common direction and degrades performance.  This suggests that
the substrate's accumulated discourse component lives almost entirely
in a single direction, consistent with Mu et al.\ \cite{mu2017all}
findings on static word embeddings.

\section{Capacity Analysis}
\label{sec:capacity}

The substrate's effective capacity for distinguishable items is bounded
by the entropy of the address space partitioned into top-$k$ activation
sets.  Following Kanerva \cite{kanerva1988sparse} and the binomial
analysis in Joglekar \cite{joglekar2015capacity}, with $M$ hard
locations and top-$k$ activation, the substrate can host approximately
$M / (2k)$ orthogonal items before crosstalk dominates.  For our
defaults ($M = 16{,}384$, $k = 64$), this gives a theoretical ceiling
of $\approx 128$ \emph{strongly orthogonal} items per role.

In practice, the substrate hosts orders of magnitude more items because
items are not strictly orthogonal---their keys overlap, their activation
sets partially share, and recall is reconstructive rather than exact.
Empirically, on TinyStories at 100K stories we observe co-occurrence
recall still discriminating at vocabulary 19{,}411
(Section~\ref{sec:results}), and IS-A recall accurate on 20+ injected
facts in the sparse bank.  The wall is graceful rather than
catastrophic: as the substrate fills, separation slowly decreases
rather than abruptly collapsing.

\paragraph{Federated learning by addition.}
Two agents instantiated with the same seed have identical hard-location
addresses;  their counter banks can be merged by direct addition:
$C_{\mathrm{merged}} = C_A + C_B$.  This implements emergent ``hive''
intelligence or Lamarckian generational inheritance via a single matrix
operation.  Capacity is bounded:  noise floor grows as $\sqrt{N}$ over
$N$ merged agents.  Destructive interference between agents with
conflicting experience is possible;  if Agent A learns
\textit{red} $\to$ reward and Agent B learns \textit{red} $\to$ punish,
their sum may interfere destructively at the red-token address.
Use-cases that benefit include swarm intelligence
\cite{indiveri2011neuromorphic,davies2018loihi}, evolutionary
populations with knowledge inheritance, and disjoint-environment
exploration where agents see different regions.

\section{Discussion}
\label{sec:discussion}

\subsection{Design philosophy: gist over exact}
The substrate's fundamental trade is exact retrieval for bounded RAM.
This is the bargain biological brains make.  A reader does not retain
every word of every book read;  they retain a fuzzy gist, with the
most-reinforced patterns preserved and the long tail blurred.  This is
implementational, not aspirational---the substrate cannot avoid this
trade because the laws of information theory \cite{shannon1948mathematical}
do not permit unbounded distinct information in finite bytes.  But the
trade is correct for the use-case:  online-learning agents do not need
perfect recall;  they need reliable retrieval of the patterns that
matter.

\subsection{Why this works without backpropagation}
Neural networks distribute representations as a byproduct of training
under a global loss.  Hyperdimensional Computing achieves the same
distribution as an \emph{algebraic property} of high-dimensional random
vectors.  Two random unit phasors in $\mathbb{C}^{512}$ have expected
cosine similarity near zero;  the ``blessing of dimensionality'' makes
distinctness free.  Combined with the SDM substrate's sparse activation
sets, the architecture obtains the distributed-representation benefits
of neural networks (graceful degradation, generalization to nearby
inputs) without needing gradient descent to discover them.

\subsection{What the no-forgetting result implies}
Section~\ref{sec:no-forgetting} reports 100\% retention of 5 IS-A facts
through 5{,}000+ distractor writes spanning four channels.  This is a
structural, not learned, property.  New writes activate different hard
locations than the original facts' locations (because the input keys
are different);  there is no shared weight matrix that gradient descent
could perturb.  The architectural design choice is the proof.
Transformers face catastrophic forgetting precisely because every
training step modifies every weight;  a flat substrate that writes
sparsely cannot exhibit this failure mode.

\section{Limitations}
\label{sec:limits}

\paragraph{Reconstructive recall.}  The substrate is gist-based, not
exact-lookup.  Inappropriate for problems requiring perfect retrieval
with unlimited RAM, such as financial transaction stores or precise
fact databases.

\paragraph{Capacity is bounded} by $M \cdot k$.  Pushing beyond
saturates the substrate and degrades separation.  Larger $M$ scales
capacity at linear RAM cost.  At extreme scales the GPU/FPGA
implementations discussed in Section~\ref{sec:related} become important.

\paragraph{Structural crosstalk} at low $M_{\mathrm{rel}}/k$ ratios
produces 5--7\% error on decoded scalars (e.g., verb coefficients) even
with sufficient writes per item.  Higher $M_{\mathrm{rel}}$ reduces it;
principled mitigations include per-channel banks or higher-rank
representations.

\paragraph{Top-$k$ activation is $O(M \cdot d)$ per query.}  Not
GPU-friendly in current form (warp divergence on
\texttt{argpartition}).  Real-time loops should cache activation sets
per common state.  Neuromorphic hardware
\cite{indiveri2011neuromorphic,merolla2014million,davies2018loihi}
provides a natural target for accelerated implementation.

\paragraph{Training-time RAM} (Python heap, PyArrow buffers when using
HuggingFace datasets) is separate from substrate RAM and can grow
during ingestion.  Inference RAM (post-checkpoint-load) is the stable,
deployable number.

\paragraph{Common-direction removal} requires a vocabulary sample for
SVD at recall time.  This is an $O(S \cdot d^2)$ operation cached
between training phases.  In a true real-time online setting it may
need to be incrementalized via streaming PCA.

\paragraph{Generation quality.}  Markov-walk generation over the
\texttt{next} role produces locally coherent text but drifts on
timescales beyond approximately 6--8 tokens.  Extension to trigrams or
4-grams via additional roles is straightforward and would address this
limitation at constant additional RAM.

\section{Related Work}
\label{sec:related}

We have referenced the most directly relevant prior art throughout
Sections~\ref{sec:background} and~\ref{sec:arch}.  We summarize the
relationships here.

\paragraph{Sparse Distributed Memory.}  Kanerva \cite{kanerva1988sparse}
and the cerebellar-modeling tradition
\cite{marr1969cerebellum,albus1971theory,eccles1967cerebellum,ito2008control}
provide the substrate primitive.  Frady and Sommer's TPAM
\cite{frady2019robust} extends SDM to complex phasor space.  This work
inherits the SDM substrate and adds multi-channel role-binding, a
compute-not-store key scheme, and dynamic mean-removal at recall.

\paragraph{VSA / HDC.}  The Plate / Gayler / Kanerva tradition
\cite{plate1995holographic,gayler2003vector,kanerva2009hyperdimensional}
provides the binding and bundling operators.  Resonator Networks
\cite{frady2020resonator,kent2020resonator2} provide efficient
factorization of bound structures, of which our cleanup procedure is a
single-pass instance.

\paragraph{Continual learning.}  EWC
\cite{kirkpatrick2017overcoming}, GEM \cite{lopez2017gradient},
MAS \cite{aljundi2018memory}, and LoRA \cite{hu2022lora} are
gradient-based mitigations of catastrophic forgetting within neural
network parameter spaces.  Our claim is that the parameter-space
problem can be sidestepped entirely by switching to a sparse-write
substrate where forgetting is structurally impossible.

\paragraph{Memory in deep architectures.}  Memory Networks
\cite{sukhbaatar2015end}, Differentiable Neural Computers
\cite{graves2016hybrid}, and Compressive Transformers
\cite{rae2020compressive} address memory in deep models but require
end-to-end gradient training.  Our approach is gradient-free and CPU
deployable.

\paragraph{Modern Hopfield networks.}  Krotov \& Hopfield
\cite{krotov2016dense} and Ramsauer et al.\ \cite{ramsauer2020hopfield}
demonstrate exponential pattern capacity in dense associative memory.
Their machinery is complementary to ours;  they could be used as
cleanup memory atop our SDM substrate.

\paragraph{Neuromorphic hardware.}  TrueNorth
\cite{merolla2014million}, Loihi \cite{davies2018loihi}, and the
broader silicon-neuron community \cite{indiveri2011neuromorphic} are
natural deployment targets;  the substrate's sparse-write semantics
match these chips' event-driven design.

\paragraph{Distinction.}  None of the above prior systems combine
zero-storage computed-identity keys with online-Hebbian VSA-bound
multi-role SDM and dynamic mean-removal cleanup in a single deployable
unit under MIT license.  The contribution is the integration plus the
empirical findings (traffic-class bank separation, $\alpha$ ablation,
no-forgetting under cross-modal flooding).

\section{Conclusion}
\label{sec:conclusion}

A brain that fits in 233~MB and absorbs frontier-scale text and pixels
while memory never grows.  Not a model.  An organism.

The substrate is biologically grounded
\cite{kanerva1988sparse,marr1969cerebellum,ito2008control}, mathematically
rigorous, and practically deployable.  It is the wrong tool for raw
spatial physics or exact lookup, and the right tool for cognitive,
episodic, symbolic, sensorimotor, and associative memory in any digital
organism that must learn online within a bounded RAM budget.  We release
\texttt{flatmem} as a standalone Python package
(\texttt{pip install flatmem}) under MIT license so that ALife
researchers, embedded-AI developers, and cognitive-modeling labs can
drop a flat substrate into their systems without porting our entire
organism stack.

Future directions include:  trigram and $n$-gram channels for improved
generation coherence;  larger and more diverse corpora to exercise the
substrate past the d=512 dictionary crossover (vocabulary $\gtrsim 50{,}000$);
GPU and neuromorphic ports
\cite{merolla2014million,davies2018loihi,indiveri2011neuromorphic} with
differentiable top-$k$ relaxation;  empirical multi-agent federated-merge
benchmarks;  and integration with reinforcement-learning algorithms via
the existing scalar-phasor encoder.

from flatmem import MultiRoleMemory

mem = MultiRoleMemory(d=512, M=16384, k=64, seed=42, M_rel=8192)

             ('manatee','mammal'),  ('penguin','bird'),

# Flood with 20 distractor IS-A facts
distractors = [('dog','mammal'), ('cat','mammal'), ('rose','flower'),
